### 背景 / 目的
現在の管理者画面は認証なしで誰でもアクセスできる。ローン申込情報（個人情報）を扱うシステムとして、Supabase Auth によるメール/パスワード認証と admin ロール管理を実装し、事務担当者のみが管理画面にアクセスできるようにする。

- 依存: #1
- ラベル: frontend, backend

### スコープ / 作業項目
- `profiles` テーブルのマイグレーション（`auth.users` 拡張・`role` カラム）
- `/admin/login` ページの実装（Supabase Auth メール/パスワード）
- ログイン後セッションを維持するコンテキスト（`AuthContext`）の実装
- `/admin` 配下の認証ガード（未認証 or admin 以外 → `/admin/login`）
- ログアウト機能の実装
- Supabase ダッシュボードで admin ユーザーを手動作成する手順をコメントに記載

### ゴール / 完了条件（Acceptance Criteria）
- [ ] `profiles` テーブルが `auth.users.id` を FK に持ち、`role = 'admin'` のレコードで管理者を識別できる
- [ ] `/admin/login` にメールアドレス・パスワード入力フォームが実装されている
- [ ] 正しい認証情報でログインすると `/admin` へリダイレクトされる
- [ ] 未認証で `/admin` にアクセスすると `/admin/login` へリダイレクトされる
- [ ] `profiles.role != 'admin'` のユーザーがアクセスしても `/admin/login` へリダイレクトされる
- [ ] 管理画面にログアウトボタンがあり、押下でセッションを破棄して `/admin/login` へ遷移する

### テスト観点
- 手動: ブラウザのシークレットモードで `/admin` へ直接アクセスしリダイレクトを確認
- 手動: 正しい認証情報でログインし管理画面が表示されることを確認
- 手動: ログアウト後に `/admin` へアクセスしリダイレクトを確認
- 検証方法: Supabase ダッシュボードの Auth セクションでセッション状態を確認

要確認事項:
- admin ユーザーの初期作成方法（Supabase ダッシュボード手動 or スクリプト）
- PoC 期間中のパスワードリセット対応の要否
