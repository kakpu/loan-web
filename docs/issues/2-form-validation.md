### 背景 / 目的
現在の申込フォーム（`CustomerFlow.tsx`）は HTML5 のネイティブバリデーションのみで、要件定義（`01_requirements.md`）が定める形式チェック（携帯電話番号 11 桁・18 歳以上など）が未実装。API-02 相当のバリデーションロジックを追加し、S01 → S02 遷移を確実にする。

- 依存: #1
- ラベル: frontend

### スコープ / 作業項目
- メールアドレス形式チェック（RFC 準拠の正規表現）
- 携帯電話番号チェック（`090`/`080`/`070` 始まり・ハイフンなし 11 桁）
- 生年月日チェック（`YYYY-MM-DD` 形式・18 歳以上）
- 必須項目の存在確認（email / phone / birthdate / desired_amount / product_name）
- バリデーションエラーをフィールド直下にインライン表示（`alert()` 廃止）
- 個人情報の `console.log` 出力を除去

### ゴール / 完了条件（Acceptance Criteria）
- [ ] メールアドレスが不正な形式の場合、フィールド直下にエラーメッセージが表示される
- [ ] 携帯電話番号が `090`/`080`/`070` 始まり・11 桁でない場合にエラーが表示される
- [ ] 生年月日で 18 歳未満を入力した場合にエラーが表示される
- [ ] 必須項目が未入力のまま「申込確定」を押すとエラーが各フィールド直下に表示される
- [ ] 全バリデーション通過後に `applications` テーブルへ INSERT し S01→S02 に遷移する
- [ ] `console.log` / `console.error` で個人情報（email・phone・birthdate）が出力されていない

### テスト観点
- 手動: 各バリデーション条件を境界値でテスト（17 歳 364 日 / 18 歳 0 日 など）
- 手動: 全フィールド正常入力で申込が完了し S02 へ遷移することを確認
- 検証方法: ブラウザ DevTools の Console タブで個人情報が出ていないことを目視確認
